PROJECT OVERVIEW
================================================================================
A professional, responsive FTP download portal for ISP customers.
- Pure HTML5 + CSS3 (no dependencies)
- WCAG 2.1 Level A accessible
- Mobile responsive (320px - 1920px)
- Sub-2-second load times
- 12 sample content items in 4 categories

STEP 1: LOCAL TESTING (5 MINUTES)
================================================================================

Option A: Python (Recommended)
  cd e:\ispdownloader
  python -m http.server 8000
  Open: http://localhost:8000

Option B: Node.js
  npx http-server
  Open: http://localhost:8080

Option C: Direct File
  Double-click index.html to open in browser

STEP 2: CUSTOMIZE FOR YOUR ISP (10 MINUTES)
================================================================================

Edit index.html and replace:

1. ISP Abbreviation
   Find:    <div class="logo-placeholder">ISP</div>
   Replace: <div class="logo-placeholder">ABC</div>

2. Portal Name
   Find:    <h1>ISP Media Vault</h1>
   Replace: <h1>ABC Media Vault</h1>

3. FTP Server Address (IMPORTANT)
   Find:    ftp://media.isp.local/
   Replace: ftp://192.168.1.100/
   (Use your actual FTP server IP or hostname)

4. Homepage Link
   Find:    https://www.example-isp.com
   Replace: https://www.yourisp.com

5. Contact Information
   Find:    support@isp.local
   Replace: support@yourisp.com

STEP 3: VERIFY FTP CONFIGURATION
================================================================================

1. Test FTP connectivity:
   - Open file manager
   - Type: ftp://192.168.1.100
   - Verify you can browse files

2. Confirm file paths match:
   - Check /Software directory exists
   - Check /Documentation directory exists
   - Check /Media directory exists
   - Check /LocalContent directory exists

3. Enable anonymous access (if needed):
   - Configure FTP server for anonymous read-only access
   - Or use token-based URLs with credentials

STEP 5: DEPLOY TO PRODUCTION
================================================================================

Option A: Apache (Linux)
  1. ssh user@server.com
  2. sudo mkdir -p /var/www/html/ispdownloader
  3. scp index.html styles.css user@server.com:/var/www/html/ispdownloader/
  4. sudo chmod 755 /var/www/html/ispdownloader
  5. Access: http://your-server/ispdownloader/

Option B: Nginx (Linux)
  1. ssh user@server.com
  2. sudo mkdir -p /usr/share/nginx/html/ispdownloader
  3. scp index.html styles.css user@server.com:/usr/share/nginx/html/ispdownloader/
  4. Configure Nginx (see DEPLOYMENT.md)
  5. Access: http://your-server/ispdownloader/

Option C: IIS (Windows)
  1. Create folder: C:\inetpub\wwwroot\ispdownloader
  2. Copy index.html and styles.css
  3. Open IIS Manager
  4. Add Website → Point to folder
  5. Access: http://localhost/ispdownloader/

Option D: Cloud (Netlify/GitHub Pages)
  1. Push files to GitHub
  2. Connect to Netlify
  3. Deploy automatically
  4. See DEPLOYMENT.md for details

CUSTOMIZATION QUICK REFERENCE
================================================================================

Change Colors (Edit styles.css):
  --primary-color: #0052cc;      (Main brand color)
  --secondary-color: #f5a623;    (Accent color)
  --success-color: #27ae60;      (Download button)

Add New Content Item:
  1. Copy <article class="content-item"> block
  2. Update: file name, size, date, description
  3. Update: FTP URL and download attribute
  4. Paste into appropriate category

Add New Category:
  1. Copy <section class="content-category"> block
  2. Update: category heading and ID
  3. Add content items
  4. Paste into appropriate location

TROUBLESHOOTING
================================================================================

Issue: Downloads don't start
  → Verify FTP server is running
  → Test FTP URL in file manager
  → Check firewall allows port 21

Issue: Page loads slowly
  → Check network connectivity
  → Verify FTP server performance
  → Test from different location

Issue: Mobile layout broken
  → Clear browser cache (Ctrl+Shift+Delete)
  → Test in Chrome DevTools device emulation
  → Check CSS file is loading

Issue: Links appear broken
  → Verify FTP server address is correct
  → Check file paths match FTP structure
  → Test URL in FTP client
